## **STEP 4: Enable GitHub Pages (1 min)**

Now make your site live:

1. **Click** the "Settings" tab (top of your repo page)
   - It's next to "Code", "Issues", "Pull requests"

2. **In the left sidebar**, scroll down and click **"Pages"**

3. **Under "Source":**
   - Click the dropdown (says "None" by default)
   - Select **"Deploy from a branch"**

4. **Under "Branch":**
   - First dropdown: Select **"main"** (or "master")
   - Second dropdown: Select **"/ (root)"**
   - Click **"Save"**

5. **Wait 30-60 seconds** for deployment

6. **Refresh the page**

You'll see a blue/green box at the top:
```
✅ Your site is live at https://yourusername.github.io/v10-truth-os/
```

## **TROUBLESHOOTING**

### **Problem: Can't see "Pages" in Settings**
**Solution:** Make sure repository is Public (not Private)

### **Problem: "Public" option is greyed out**
**Solution:** You need to verify your email address first

### **Problem: Site shows 404 error**
**Solution:** 
- Wait 2-3 minutes (GitHub Pages takes time to deploy)
- Make sure `index.html` is in the root (not in a subfolder)
- Refresh browser with Ctrl+F5

### **Problem: CSS/styling broken**
**Solution:**
- Check browser console (F12) for errors
- Make sure all files uploaded correctly
- File names are case-sensitive

### **Problem: Files won't upload**
**Solution:**
- Try uploading in smaller batches (5 files at a time)
- Check file size (GitHub has 100MB file limit)
- Try refreshing GitHub page and trying again
